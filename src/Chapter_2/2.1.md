# অধ্যায় ২: এনভায়রনমেন্ট সেটআপ (Windows)

**ESP মাইক্রোকন্ট্রোলার প্রোগ্রামিং-এর জন্য বেশ কিছু প্রচলিত পদ্ধতি রয়েছে। যেমন:**
1. ESP-IDF Command Prompt ব্যবহার করে
2. Espressif-IDE ব্যবহার করে
3. Arduino Framework ব্যবহার করে
4. ESP-IDF VSCode Extension ব্যবহার করে

তবে উপরের প্রতিটি পদ্ধতিরই কিছু না কিছু drawback (সীমাবদ্ধতা) রয়েছে। আমি চেষ্টা করব **ESP-IDF Framework + VS Code**-এর সংমিশ্রণে একটি ক্লিন-ইন্টারফেস ব্যবহার করে ESP32 প্রোগ্রামিং শেখানোর, যাতে একই সাথে **Easy Interface** এবং **Latest Espressif Release** ব্যবহারের এক দুর্দান্ত অভিজ্ঞতা পাওয়া যায়।


---

## ২.১ এই অধ্যায়ে আমরা কী শিখব?

প্রথম অধ্যায়ে আমরা জেনেছি *Embedded System*, *ESP32*, এবং *ESP-IDF* কী। এখন হাতে-কলমে কাজ শুরু করার পালা। এই অধ্যায়ে আমরা `Windows Operating System (OS)` চালিত কম্পিউটারে এমন একটি এনভায়রনমেন্ট তৈরি করব, যেখানে **ESP-IDF Command Prompt** ব্যবহার করে প্রোজেক্ট তৈরি করা, বিল্ড করা, এবং বোর্ডে আপলোড (Flash) করার মতো জটিল কাজগুলো খুব সহজেই করা যায়।

> **নোট**: Windows এবং macOS-এ ইনস্টলেশনের ধাপ কিছুটা ভিন্ন। এই অংশে আমরা **Windows**-এর জন্য পুরো প্রক্রিয়াটি দেখব, এবং পরের অংশে **macOS**-এর জন্য আলাদাভাবে এনভায়রনমেন্ট তৈরি করা শিখব। আপনি যদি macOS ব্যবহার করেন, তাহলে সরাসরি পরের অধ্যায়ে চলে যেতে পারেন।

**আমরা ধাপে ধাপে যা করব:**
1. Visual Studio Code (VS Code) ইনস্টল করা
2. ESP-IDF ফ্রেমওয়ার্ক ইনস্টল করা
3. New Project তৈরি এবং বিল্ড করা 
4. ESP32-তে প্রোগ্রাম আপলোড করা 

> **নোট**: ESP-IDF Framework-এর ভার্সন নম্বর মাঝেমধ্যে আপডেট হয়। এই অধ্যায়টি লেখার সময় (২০২৬) Espressif-এর অফিসিয়াল ওয়েবসাইটে ESP-IDF এর সর্বশেষ সংস্করণ ESP-IDF v5.5.4 এর screenshot ব্যবহার করা হয়েছে। ভবিষ্যতে ভার্সন নম্বরে সামান্য পরিবর্তন দেখলে ঘাবড়াবেন না — মূল ধারণাটি সবসময় একই থাকবে।

## ২.৫ ধাপ ৩: New Project তৈরি এবং বিল্ড করা  
এতক্ষণে `VS Code` এবং `ESP-IDF` ফ্রেমওয়ার্ক — সবকিছু ইনস্টল করা হলো। এখন আমরা একটি সাধারণ কম্পিউটার প্রোগ্রাম (হ্যালো ওয়ার্ল্ড) `create` এবং `run` করে দেখবো যে সবকিছু ঠিকঠাক কাজ করছে কিনা। যেকোনো প্রোগ্রামিং ভাষা শেখার ক্ষেত্রে এটি একটি common প্রোগ্রাম।

## ২.৬ ধাপ ৩: ESP32-তে প্রোগ্রাম আপলোড করা 
আপনার কাছে থাকা ESP32 Microcontroller-এ প্রোগ্রাম আপলোড করতে, নিচের ধাপগুলো অনুসরণ করুন: 
1. প্রোগ্রাম আপলোড করতে সর্বপ্রথম ESP32 Dev Kit V1 কে USB Cable এর সাহায্যে কম্পিউটারের সাথে connect করতে হবে। 

2. ESP বোর্ডে থাকা USB-to-Serial চিপ কম্পিউটারের সাথে যোগাযোগের জন্য COMxx Port তৈরি করে। বোর্ডটি কম্পিউটারের কোন COMxx পোর্টে কানেক্ট হয়েছে, তা Device Manger থেকে খুব সহজেই চেক করা যায়। এ জন্য কম্পিউটারের Start বাটন ক্লিক করে, Device Manager লিখে সার্চ করে App টি সিলেক্ট করতে হবে। 

<figure align="center">
    <img src="../../Image/CH2/2-6-1-Device-Manager.png" width="50%">
    <figcaption>Figure: Device Manager</figcaption>
</figure>

ESP বোর্ডে USB to Serial Communication এর জন্য সাধারণত **CP2102** অথবা **CH340** চিপ ব্যবহার করা হয়। 

আপনার বোর্ডে কোন চিপ ব্যবহার করা হয়েছে তা বোর্ডের ছোট IC-গুলোর গায়ে লেখা নাম পড়ে নিতে পারেন, অথবা Device Manager-এর **Ports (COM & LPT)** সেকশনে বোর্ড কানেক্ট করার পর নতুন কোন COM পোর্ট (যেমন `COM5`) তৈরি হচ্ছে সেটা দেখেও নিশ্চিত হতে পারেন। 

<figure align="center">
    <img src="../../Image/CH2/2-6-2-Device-Manager-COM5.png" width="50%">
    <figcaption>Figure: USB to UART Bridge (COM5)</figcaption>
</figure>

যেমন আমরা কম্পিউটারে **CP210x USB to Serial** ড্রাইভার চিপ `COM5` পোর্ট তৈরি করেছে। এই পোর্ট-নম্বরটি খুবই গুরুত্বপূর্ণ, এটি ব্যবহার করেই আমরা পরবর্তীতে ESP-তে প্রোগ্রাম আপলোড বা সিরিয়াল-ডেটা প্রিন্ট করার মত কাজ গুলো করতে পারবো। 

3. ESP32 Dev Kit V1-এ Hello-World প্রোজেক্টটি binary file flash করার জন্য নিচের কমান্ডটি run করুন। 

```
idf.py -p com5 flash 
```

যদিও আমি বোঝার সুবিধার্থে ভেঙ্গে ভেঙ্গে কমান্ড দিয়েছি, তবে আপনি ইচ্ছা করলে সরাসরি নিচের মত করে single line command লিখতে পারেন, এতে করে Program Upload হয়ে সরাসরি Terminal-এ  Serial Data প্রিন্ট হওয়া শুরু হবে।
